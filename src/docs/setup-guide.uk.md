# Інструкція з налаштування

## Що потрібно

- **Bambu Lab A1 Mini** (модель принтера N1)
- **Bambu Studio** або **Orca Slicer**
- SD-карта або мережеве підключення до принтера

---

## Крок 1: Наріжте модель

1. Відкрийте модель у **Bambu Studio** або **Orca Slicer**
2. Налаштуйте параметри друку (висота шару, заповнення, підтримки тощо)
3. Якщо у вас кілька столів — розмістіть моделі як потрібно
4. Натисніть **Slice** і дочекайтесь завершення
5. Збережіть проект як `.3mf` файл

> **Важливо:** Файл `.3mf` повинен містити G-code (бути нарізаним). Сирий файл проекту без нарізки не підійде.

---

## Крок 2: Згенеруйте SWAP файл

1. Відкрийте **swapHub**
2. Перетягніть `.3mf` файл у зону завантаження
3. Налаштуйте кожен стіл:
   - **Увімк/Вимк** — включити або виключити стіл з черги
   - **Повтори** — скільки разів друкується стіл за один цикл
   - **Перетягування** — змініть порядок друку перетягуванням карток
4. Встановіть **Повтори циклу** — скільки разів повторюється вся черга
5. (Опціонально) Налаштуйте опції:
   - **Вимкнути Mech Mode fast check** — пропустити механічну перевірку між столами (залишає стартову перевірку на першому столі)
     - **Повністю (включаючи першу)** — додаткова галочка, яка прибирає перевірку взагалі з усіх столів, включаючи перший
   - **Вимкнути Dynamic Flow Calibration** — пропустити повторну калібрацію потоку
6. Оберіть **обкладинку** з галереї (мініатюра для дисплея принтера)
7. Натисніть **Згенерувати SWAP файл**

---

## Крок 3: Відправте на принтер

### Варіант А: SD-карта

1. Скопіюйте згенерований `.swap.3mf` на SD-карту
2. Вставте SD-карту у принтер
3. Оберіть файл у меню принтера і запустіть друк

### Варіант Б: Bambu Studio / Orca Slicer

1. Відкрийте `.swap.3mf` у Bambu Studio або Orca Slicer
2. Відправте на принтер через мережу (LAN або хмара)

---

## Як працює черга

Черга друку формується з налаштованих столів:

```
Для кожного повтору циклу:
  Для кожного увімкненого столу (по порядку):
    Друкувати стіл × кількість повторів
```

**Приклад:**
- Стіл **A** — 3 повтори
- Стіл **B** — 1 повтор
- Повтори циклу — 2

Результат: `A A A B A A A B`

## Структура G-code

Згенерований `.swap.3mf` містить один об'єднаний G-code файл (`plate_1.gcode`):

1. **Стартовий G-code** з першого столу (хомінг, нагрів, вирівнювання столу)
2. Для кожного столу в черзі:
   - G-code столу (друк)
   - Swap-послідовність (очищення столу, команди зміни)
3. **Фінальний G-code** з останнього столу (вимкнення двигунів, паркування)

На останньому столі порядок: друк → зміна столу → фінішний блок (FIN). Тобто стіл змінюється **до** вимкнення двигунів, щоб можна було забрати готовий друк.

### Swap-послідовність між столами

Між кожним друком вставляється такий G-code:

```gcode
; ===== swap: plate X =====
M400 S1        ; Дочекатись завершення рухів
G28 X           ; Хомінг осі X
M400 S1        ; Дочекатись
; ... послідовність очищення столу ...
```

## Розмітка G-code у слайсері

Щоб опції **Вимкнути Mech Mode fast check**, **Вимкнути Dynamic Flow Calibration** та коректне відсікання фінішного блоку працювали, G-code повинен містити спеціальні маркери-коментарі. Їх потрібно додати у налаштуваннях слайсера (Bambu Studio або Orca Slicer) у відповідних секціях G-code.

### MMFC — Mech Mode Fast Check

Оберніть блок механічної перевірки у стартовому G-code маркерами:

```gcode
G1 Z5 F3000
;===START: MMFC===
  G1 X90 Y-1 F30000
  M400 P200
  M970.3 Q1 A7 K0 O2
  M974 Q1 S2 P0

  G1 X90 Y0 Z5 F30000
  M400 P200
  M970 Q0 A10 B50 C90 H15 K0 M20 O3
  M974 Q0 S2 P0
;===END: MMFC===
M975 S1
G1 F30000
G1 X-1 Y10
G28 X ; re-home XY
```

swapHub шукає ці маркери і видаляє весь блок між ними. Без маркерів опція не матиме ефекту.

**Де додати:** Стартовий G-code принтера (Printer Start G-code) у налаштуваннях слайсера.

### DFC — Dynamic Flow Calibration

Оберніть блок калібрації потоку маркерами:

```gcode
;M1002 set_flag extrude_cali_flag=1
;===START: DFC===
M1002 judge_flag extrude_cali_flag
M622 J1
    M1002 gcode_claim_action : 8
    
    M400
    M900 K0.0 L1000.0 M1.0
    G90
    M83
    G0 X68 Y-4 F30000
    G0 Z0.3 F18000 ;Move to start position
    M400
    G0 X88 E10  F{outer_wall_volumetric_speed/(24/20)    * 60}
    G0 X93 E.3742  F{outer_wall_volumetric_speed/(0.3*0.5)/4     * 60}
    G0 X98 E.3742  F{outer_wall_volumetric_speed/(0.3*0.5)     * 60}
    G0 X103 E.3742  F{outer_wall_volumetric_speed/(0.3*0.5)/4     * 60}
    G0 X108 E.3742  F{outer_wall_volumetric_speed/(0.3*0.5)     * 60}
    G0 X113 E.3742  F{outer_wall_volumetric_speed/(0.3*0.5)/4     * 60}
    G0 Y0 Z0 F20000
    M400
    
    G1 X-13.5 Y0 Z10 F10000
    M400
    
    G1 E10 F{outer_wall_volumetric_speed/2.4*60}
    M983 F{outer_wall_volumetric_speed/2.4} A0.3 H[nozzle_diameter]; cali dynamic extrusion compensation
    M106 P1 S178
    M400 S7
    G1 X0 F18000
    G1 X-13.5 F3000
    G1 X0 F18000 ;wipe and shake
    G1 X-13.5 F3000
    G1 X0 F12000 ;wipe and shake
    G1 X-13.5 F3000
    M400
    M106 P1 S0

    M1002 judge_last_extrude_cali_success
    M622 J0
        M983 F{outer_wall_volumetric_speed/2.4} A0.3 H[nozzle_diameter]; cali dynamic extrusion compensation
        M106 P1 S178
        M400 S7
        G1 X0 F18000
        G1 X-13.5 F3000
        G1 X0 F18000 ;wipe and shake
        G1 X-13.5 F3000
        G1 X0 F12000 ;wipe and shake
        M400
        M106 P1 S0
    M623
    
    G1 X-13.5 F3000
    M400
    M984 A0.1 E1 S1 F{outer_wall_volumetric_speed/2.4} H[nozzle_diameter]
    M106 P1 S178
    M400 S7
    G1 X0 F18000
    G1 X-13.5 F3000
    G1 X0 F18000 ;wipe and shake
    G1 X-13.5 F3000
    G1 X0 F12000 ;wipe and shake
    G1 X-13.5 F3000
    M400
    M106 P1 S0
M623 ; end of "draw extrinsic para cali paint"
;===END: DFC===

;Load DFC settings from memory
M622 J0
    M984 A0.1 E1 S0 F{outer_wall_volumetric_speed/2.4} H[nozzle_diameter]
M623
```

Додатково, swapHub шукає рядок `;M1002 set_flag extrude_cali_flag=1` і додає після нього `M1002 set_flag extrude_cali_flag=0` щоб скасувати прапорець калібрації на не-перших столах.

**Де додати:** Стартовий G-code принтера (Printer Start G-code) у налаштуваннях слайсера.

### FIN — Фінішний блок

Оберніть фінішний G-code (охолодження, паркування, вимкнення) маркерами (додайте в самий кінець коду):

```gcode
;===START: FIN===
M400 ; wait all motion done
M18 X Y Z ; disable steppers, except E
;===END: FIN===
```

swapHub автоматично видаляє фінішний блок з **усіх столів крім останнього** — щоб принтер не вимикав нагрів і не паркувався між друками, а виконав фініш тільки після останнього столу в черзі.

**Де додати:** Кінцевий G-code принтера (Printer End G-code) у налаштуваннях слайсера.

> **Важливо:** Якщо маркери відсутні в G-code, відповідні опції swapHub просто не матимуть ефекту — помилки не буде, але блоки не будуть розпізнані й видалені.

## Вирішення проблем

### "Цей файл нарізано для принтера X"

SwapMod працює тільки з **Bambu Lab A1 Mini (N1)**. Переконайтесь, що ви обрали A1 Mini як принтер у слайсері перед нарізкою.

### Файл не відкривається / "Неможливо прочитати файл"

Переконайтесь, що файл — валідний `.3mf` архів. Спробуйте заново експортувати зі слайсера.

### "Не знайдено нарізаних столів"

Файл містить геометрію, але не має G-code. Спершу **наріжте** модель, потім збережіть.

### Друк зупиняється під час swap

- Переконайтесь, що стіл чистий і адгезія хороша
- Спробуйте вимкнути **Mech Mode fast check** якщо принтер надто довго зупиняється між столами. Якщо стартова перевірка також заважає — увімкніть додатково опцію **Повністю (включаючи першу)**
- Перевірте, що філамент завантажений у правильні слоти AMS
